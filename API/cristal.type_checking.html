cristal.type_checking.check_types(conditions: dict[str, Callable[[Any], bool]] | None = None) → Callable#

Decorator to check types of function arguments.

Parameters:

conditions (dict[str, Callable[[Any], bool]] | None, optional) – Custom conditions for argument validation, by default None

Returns:

The decorated function with type checks applied.

Return type:

Any

Raises:

TypeError – If an argument does not match its expected type or fails a custom condition.

Example

This will raise a TypeError if x is not float:

@check_types()
def my_function(x: float) -> None:
    pass

This will raise a TypeError if a is not an int greater than 0 or if b is not a str or None:

@check_types({"a": lambda x: x > 0})
def my_function(a: int, b: str | None = None) -> None:
    pass

Notes

Warning

The decorator does not check for the return type of the function.

Warning

The decorator does not work with parametrized generic types (e.g., List[int]). This will raise a TypeError if used with such types when using my_function:

@check_types()
def my_function(my_list: list[int]) -> None:
    pass

Instead, use check_all_of_type() to check if all elements in a list are of a specific type:

@check_types({"my_list": lambda x: check_all_of_type(x, int)})
def my_function(my_list: list) -> None:
    pass
